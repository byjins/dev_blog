# 블로그를 만든 이유
취준생 시절에는 살아남기 위해 처절하게 공부하고.. velog, notion 등을 이용하여 정리하고 그랬는데.. 취업 이후엔 너무 편해졌던 것일지 지식??을 얻기 위해 어떠한 노력도 하지 않았습니다..
> 반성중입니다..

그래서 `2025년도의 목표`가 하나 생겼습니다. 1일 1커밋의 수준까진 아니더라도 꾸준히 기술의 최신 트렌드를 파악하고, 하고있는 직무(프론트엔드)에서 조금 더 나은 개발자가 되기 위한 공부를 해야겠다라고 생각했습니다.

일단 가장 먼저 생각난건 `NextJS`로 블로그를 만들어보자! 였습니다.
물론 좋은 블로그들`(Velog, Tistory 등..)`은 많지만.. 그래도 개발자인데! 한번 만들어보는게 어떨까 했습니다.
> 내가 만든 나만의 블로그라니.. 벌써 설렙니다!

그리고 재직중인 회사에선 NextJS를 사용하지 않는데 요즘 여러 회사들은 우대사항에 상당히 많더라구요.

물론 NextJS로 블로그를 만든다고 `NextJS`에 대해 척척박사가 되진 않겠지만, 그래도 블로그를 만들어보며 Next와 조금은 친해질 수 있지 않을까요? `원하던 공부는 덤`


## 만드는 과정
일단 저는 Next.js에 대해 아무것도 모르기 때문에 공식문서를 봤으며 Next.js로 블로그를 만들기 위해 여러 개발 블로그를 참고했습니다.
1. Next.js 프로젝트 생성 (app router)
```
npx create-next-app@latest
```
2. Vercel로 프로젝트 연동!
> 프로젝트를 연동하면 바로 배포가 되는데 이때 배포된 경로를 접속했을 때 기본 템플릿 화면이 나온다면 성공입니다!

3. 내가 생각한 화면 및 디렉터리 구조 별로 파일 생성하기!
> [Project Structure] (https://nextjs.org/docs/app/getting-started/project-structure)
[Layout and Pages] (https://nextjs.org/docs/app/getting-started/layouts-and-pages)

4. 개발 마구마구 하기
> Next.js 개발 블로그 만들기에 대한 내용은 Google에 검색하면 엄청 많이 나옵니다!
일일히 설명하기는 너무 내용이 많기도하고 참고하면 금방 할 수 있습니다!

## 블로그 기능
- 게시글 목록/상세
- 다크모드
- ToC
- `카테고리 및 태그 검색 기능 (미개발)`
- giscus(github) 댓글 기능!

이 중에서 검색 기능을 제외하곤 전부 적용했습니다!
정말 전부 처음해보는 것들이라 시간이 좀 걸린 것 같습니다. 근데 어렵진 않았습니다. 요즘은 워낙 정보도 많을 뿐더러 상당히 적용하기 쉽더군요!!

## 개선사항
- SEO
- 도메인 적용
- 블로그 목록 고민 중 (페이징 vs 무한스크롤)
- 카테고리 및 태그 검색 기능
- 조회수 기능 (할지말지 고민중..)

일단 `SEO`는 빠르게 적용해서 한번 효과를 보고싶습니다!
`SEO`는 마케팅이라는 글을 본 적이 있는데 한 번 블로그 만든 김에 `퍼스널 브랜딩`도 되면서 여러 사람들과 정보를 공유하고 싶은 마음입니다.
